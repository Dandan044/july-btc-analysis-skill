# july-btc-analysis

> BTC 技术分析技能 - 定时日报 + 警报即时分析 + 交易周期管理

专为加密货币分析智能体设计的 BTC 分析技能。提供完整的分析工作流，从数据获取到报告生成，从警报监控到交易建议管理。

---

## ✨ 亮点

| 特性 | 说明 |
|------|------|
| **⏰ 定时日报** | 每天 9:00/21:00 自动执行分析，发送飞书报告 |
| **⚡ 即时分析** | 警报触发时立即分析，响应市场变化 |
| **🔄 SPAWN 隔离** | 每次分析在新会话执行，避免上下文干扰 |
| **📋 周期管理** | 交易建议生命周期追踪，自动归档已关闭周期 |
| **🛠️ 多维警报** | 不只价格！支持交易量、波动率、情绪等多维度监控 |
| **🔥 热加载** | 警报规则无需重启引擎，自动加载新规则 |
| **📊 多数据源** | CryptoCompare + Binance，支持价格、OI、资金费率、多空比等 |

## 🚀 快速开始

### 1. 安装技能

```bash
# 从 ClawHub 安装（推荐）
clawhub install july-btc-analysis

# 或从 Git 克隆
git clone https://github.com/xxx/july-btc-analysis.git
```

### 2. 启动警报器

```bash
pm2 start skills/btc-alert/engine.js --name btc-alert
pm2 save
```

### 3. 配置定时任务

在 OpenClaw 中配置 cron：
```json
{
  "schedule": { "kind": "cron", "expr": "0 9,21 * * *", "tz": "Asia/Shanghai" },
  "payload": { "kind": "agentTurn", "message": "[SPAWN_DAILY_REPORT]" }
}
```

### 4. 配置飞书接收

在 `credentials.json` 中配置目标用户：
```json
{
  "feishu": { "targetOpenId": "ou_xxx" }
}
```

## 📋 交易周期系统

每个分析周期独立管理交易建议：

```
新周期开启 → 给出建议 → 监控止盈止损 → 全部关闭 → 自动归档
```

建议状态：
- `pending_entry` → 等待入场（警报监控）
- `open` → 持仓中（监控止盈止损）
- `closed` → 已平仓

---

## 🛠️ 警报类型

不只是价格警报！支持多维度监控：

- **价格警报**：支撑/阻力位突破
- **交易量异动**：交易量异常放大
- **波动率收窄**：变盘前夕预警
- **情绪极端**：FGI < 10 或 > 80
- **定时器警报**：计划入场时间提醒

每个方向最多 1 个警报，避免冗余。
