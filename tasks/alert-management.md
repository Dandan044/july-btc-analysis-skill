# 警报器管理任务

此任务在日报或即时分析任务**完成后**被调用，负责管理警报器规则。

**注意**：七月具有当前分析的上下文，可以直接使用分析结果（趋势判断、关键位置等）进行警报管理。

---

## 执行步骤

### 1. 查看当前规则

查看 `skills/btc-alert/rules/` 文件夹，查看所有活跃规则：

```bash
ls -la skills/btc-alert/rules/
```

对每个规则文件，读取其 `name` 和 `lifetime()` 状态。

### 2. 评估规则状态

根据本次分析结果和当前规则，决定操作：

| 情况 | 操作 |
|------|------|
| 规则目标已失效（如压力位已突破） | 归档到 `rules-archive/` |
| 规则已过期（lifetime 返回 expired） | 归档到 `rules-archive/` |
| 经过分析规则也不再适用但仍在活跃 | 归档活跃规则 |
| 分析发现新的关键位置 | 创建新规则 |
| 规则仍在有效期内且目标有效 | 保持不变 |

### 3. 归档过期规则

将不再需要的规则文件移动到归档目录：

```bash
mv skills/btc-alert/rules/<rule-name>.js skills/btc-alert/rules-archive/
```

记录到日志：`logs/alert-management.log`

```
[YYYY-MM-DD HH:mm:ss] 归档规则: <rule-name>.js | 原因: xxx
```

### 4. 设定新规则

根据分析结论，设定新警报。参考 `tasks/set-alert.md` 的规范。

**⚠️ 核心原则（必须遵守）**

**原则一：单方向单一价格警报**

每个方向（向上/向下）**最多只保留一个价格警报**。这意味着：
- 向上方向（阻力位突破）：最多 1 个
- 向下方向（支撑位跌破）：最多 1 个
- 总计最多 2 个价格警报

**原则二：创造性警报设计**

不要局限于简单的"价格到达目标位就触发"。要充分利用警报器的灵活性：

| 警报类型 | 适用场景 | 优势 |
|---------|---------|------|
| **延迟触发** | 突破后确认、避免假突破 | 观察突破后的走势稳定性 |
| **定时器** | 计划入场时间、等待事件发酵 | 不依赖市场数据，纯时间驱动 |
| **交易量异动** | 大资金进出、市场活跃度变化 | 提前捕捉价格变动前的资金动向 |
| **振幅/波动率** | 剧烈波动、变盘前夕 | 捕捉市场异常状态 |
| **持仓量变化** | OI增减、多空博弈 | 衍生品市场信号 |
| **多空比变化** | 散户情绪反转 | 交易侧数据，更敏感 |
| **Taker买卖比** | 主动买/卖力量 | 实时交易意愿 |
| **资金费率** | 多空付费压力 | 衍生品市场情绪 |
| **...** | **发散思考** | **不限于上述示例** |

**每次警报管理时，必须主动问自己：**
1. 除了价格，还有什么维度值得监控？
2. 当前市场有什么异常现象可能被价格警报忽略？
3. 是否需要延迟确认而非立即触发？

**原则三：禁止使用恐惧贪婪指数作为警报触发条件**

❌ **FGI 不适合作为警报监控值**，原因：
- 更新频率：一天才更新一次，无法捕捉日内变化
- 警报器需要高频数据（分钟级），FGI 无法满足
- FGI 应作为**分析参考**，而非**警报触发条件**

⚠️ 已设置的 FGI 警报必须删除。警报设计时不要包含 FGI 相关的触发逻辑。

当需要设定新警报时：
1. 确定方向（向上/向下）
2. 如果该方向已有警报，比较目标价位：
   - 新目标更优 → 归档旧规则，创建新规则
   - 旧目标仍有效 → 保持不变
3. "更优"的定义：
   - 向上方向：新目标更接近当前价格且仍具阻力意义
   - 向下方向：新目标更接近当前价格且仍具支撑意义

**示例**：
- 当前价格 84000
- 已有警报：阻力 92000（向上）、支撑 80000（向下）
- 分析发现新阻力 86000 → 更近的阻力更有意义 → 归档 92000，创建 86000
- 分析发现新支撑 82000 → 更近的支撑更有意义 → 归档 80000，创建 82000

---

**⚠️ 进阶：非价格类警报**

除了支撑/阻力位的价格警报，系统还支持监控**交易量、波动率、情绪指标**等维度。

**规则**：非价格类警报**最多 2 个**，与价格警报独立计算。

**何时设定非价格类警报？**

每次警报管理时，必须主动评估是否需要设定此类警报。判断依据：

| 市场现象 | 可能的警报类型 |
|---------|---------------|
| 交易量异常放大或萎缩 | 交易量异动警报 |
| 价格在窄幅区间长时间震荡 | 波动率收窄警报（可能预示突破） |
| 短时间内剧烈波动 | 振幅警报 |
| 连续多根同向K线 | 趋势加速警报 |
| 持仓量OI快速增加或减少 | 持仓量变化警报 |
| 多空比发生明显反转 | 多空比反转警报 |
| Taker买卖比偏离常态 | Taker买卖比警报 |
| 资金费率异常（过高或过低） | 资金费率警报 |
| ... | **发散思考，不限于上述示例** |

**⚠️ 重要：发散思维**

以上仅为示例，**切勿局限于这些场景**。每次分析时，应主动思考：

- 当前市场有什么**异常或值得关注的现象**？
- 这些现象如果**持续或反转**，会对持仓产生什么影响？
- 是否有**其他维度**的数据可以提前预警？

**警示**：不要每次只设定价格警报。如果市场处于低波动期，价格警报可能长期不触发，但交易量、波动率的变化可能在提前发出信号。忽略这些信号可能导致错过关键转折点。

**非价格警报示例**：

1. **交易量异动警报**：当小时交易量超过 30 日小时交易量均值的 2 倍时触发
2. **振幅警报**：当 1 小时振幅超过 3% 时触发
3. **波动率收窄警报**：当连续 6 小时波动率递减时触发（可能预示大行情）
4. **连阴/连阳警报**：当连续 6 根同向 K 线时触发
5. **持仓量变化警报**：当 OI 24小时内增减超过 10% 时触发
6. **多空比反转警报**：当多空比从 <1 反转为 >1.2，或从 >1 反转为 <0.8 时触发
7. **资金费率警报**：当资金费率超过 0.01% 或低于 -0.01% 时触发

以上仅为示例，根据市场实际情况可以制定更多样的警报。

---

**规则默认设定**：
- 触发动作：执行"即时分析任务"
- 冷却时间：至少 1 小时
- 有效期：根据分析结果设定，或一次性触发（触发后即归档）
