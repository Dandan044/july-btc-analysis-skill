# 实盘持仓同步任务

此任务从 OKX 实盘获取 BTC 逐仓持仓数据，同步到本地持仓文件。

---

## 触发方式

- 由工作流路由调用（阶段一步骤 2.5）
- 或由其他工作流显式调用

---

## 输入参数

调用此任务时，需提供以下参数：

| 参数 | 说明 | 示例 |
|------|------|------|
| 周期文件夹路径 | 当前活跃周期文件夹 | `active/cycle-20260419-001` |
| 日志文件路径 | 日报流程日志文件 | `logs/daily-report-process.log` |

---

## 输出产物

| 文件 | 路径 | 说明 |
|------|------|------|
| 持仓文件 | `active/cycle-*/positions.json` | BTC 逐仓持仓数据 |

---

## 执行步骤

### 1. 获取实盘持仓数据

**⚠️ 严格约束：只能获取 BTC-USDT-SWAP 逐仓仓位，绝对不能同步全仓仓位**

- 即使 API 返回了 BTC 全仓持仓，也必须忽略
- 如果没有 BTC 逐仓持仓，`当前持仓` 直接置为空数组 `[]`
- 不能将全仓数据（无论是 BTC 还是其他币种）写入 positions.json

**使用 okx 工具获取当前持仓、止盈止损订单和账单记录：**

```bash
# 获取当前持仓（必须筛选 BTC-USDT-SWAP 逐仓）
okx-proxy.sh --profile live account positions --instId SWAP --json

# 获取委托订单（止盈止损）
okx-proxy.sh --profile live swap orders --instId BTC-USDT-SWAP --json
okx-proxy.sh --profile live swap algo orders --instId BTC-USDT-SWAP --json

# 获取账单记录（当前持仓的操作历史）
okx-proxy.sh --profile live account bills --instType SWAP --ccy USDT --limit 100 --json
```

**⚠️ 数据筛选规则（执行时必须应用）：**

| 步骤 | 筛选条件 | 说明 |
|------|---------|------|
| 1 | `instId` = `BTC-USDT-SWAP` | 只保留 BTC 合约 |
| 2 | `tdMode` 或 `mgnMode` = `isolated` | 只保留逐仓模式 |
| 3 | 忽略全仓数据 | 即使有全仓仓位也忽略 |

**⚠️ 绝对禁止：将全仓数据写入 positions.json**

---

**⚠️ 止盈止损订单获取方式**

| 订单类型 | API 命令 | 订单类型字段 | 说明 |
|---------|---------|------------|------|
| **止盈订单** | `swap orders` | `ordType: "limit"` | 普通限价平仓订单 |
| **止损订单** | `swap algo orders` | `ordType: "conditional"` | 条件触发订单 |

**止盈订单特征：**
- `ordType` = `limit`
- `reduceOnly` = `true`
- `px` 字段为止盈价格
- 可通过 `linkedAlgoOrd.algoId` 关联止损订单

**止损订单特征：**
- `ordType` = `conditional`
- `slTriggerPx` 字段为止损触发价
- `slOrdPx` = `-1` 表示市价执行

### 3. 持仓同步逻辑

**同步流程（每次覆写）：**

```
1. 获取 OKX 实盘当前持仓数据
2. 筛选 BTC 逐仓仓位（instId = BTC-USDT-SWAP, mgnMode = isolated）
3. 检查持仓文件是否存在
   - 若存在，读取旧持仓 → 判断是否有仓位变化
   - 若旧持仓有仓位，新持仓为空 → 检测到平仓
4. 若检测到平仓，查询平仓记录：
   - 从旧持仓获取 持仓ID
   - 查询账单/历史持仓，找到最后平仓记录
   - 填充 最近平仓 字段
5. 获取止盈止损订单（swap orders + swap algo orders）
6. 获取账单记录，筛选当前持仓的操作历史
7. 计算汇总数据
8. 覆写持仓文件
```

---

**⚠️ 平仓检测逻辑**

**检测条件（同时满足）：**

| 条件 | 判断方式 |
|------|---------|
| 持仓文件已存在 | `positions.json` 文件存在 |
| 旧持仓有仓位 | 旧文件 `当前持仓` 数组不为空 |
| 新持仓为空 | API 返回的 BTC 逐仓持仓为空 |

**满足以上条件 → 本周期曾开仓现已平仓**

---

**平仓记录查询：**

```bash
# 查询历史持仓（已平仓）
okx-proxy.sh --profile live account positions-history --instId BTC-USDT-SWAP --json

# 或查询账单中的平仓记录（type=2, 平仓方向与持仓相反）
okx-proxy.sh --profile live account bills --instId BTC-USDT-SWAP --ccy USDT --limit 50 --json
```

**从历史持仓 API 获取平仓信息：**

| 字段 | 中文字段名 | 说明 |
|------|----------|------|
| `posId` | 持仓ID | 与旧持仓ID匹配 |
| `closeAvgPx` | 平仓均价 | 最后成交均价 |
| `uTime` | 平仓时间 | 最后更新时间 |
| `pnl` | 盈亏 | 本次平仓盈亏 |
| `fee` | 手续费 | 平仓手续费 |

**平仓类型判断：**

| 判断方式 | 平仓类型 |
|---------|---------|
| 平仓价接近旧止盈价 | 止盈触发 |
| 平仓价接近旧止损价 | 止损触发 |
| 其他 | 手动平仓 |

---

**每次覆写原则：交易所数据是权威来源，平仓信息需要从旧持仓 + API 组合获取。**

---

**委托订单筛选：**

| 订单类型 | API 命令 | 筛选条件 |
|---------|--------|---------|
| **止盈订单** | `swap orders` | `instId=BTC-USDT-SWAP`, `tdMode=isolated`, `reduceOnly=true`, `ordType=limit` |
| **止损订单** | `swap algo orders` | `instId=BTC-USDT-SWAP`, `tdMode=isolated`, `ordType=conditional` |

**止盈订单特征：**
- `ordType` = `limit`（限价订单）
- `px` 字段 = 止盈价格
- 可通过 `linkedAlgoOrd.algoId` 关联止损订单ID

**止损订单特征：**
- `ordType` = `conditional`（条件订单）
- `slTriggerPx` 字段 = 止损触发价
- `slOrdPx` = `-1` 表示市价执行

**订单匹配到仓位：**
- 通过 `posSide`（持仓方向）匹配
- 通过 `sz`（数量）匹配仓位张数

---

**API 数据映射：**

| OKX API 字段 | 中文字段名 | 说明 |
|------------|----------|------|
| `posId` | 持仓ID | 唯一标识 |
| `instId` | 合约 | BTC-USDT-SWAP |
| `tdMode` 或 `mgnMode` | 保证金模式 | isolated |
| `posSide` | 持仓方向 | long/short |
| `pos` | 持仓张数 | 当前持仓量 |
| `notionalUsd` | 名义价值USD | 名义仓位价值 |
| `avgPx` | 平均入场价 | 开仓均价 |
| `upl` | 未实现盈亏 | 当前浮盈浮亏 |
| `realizedPnl` | 已实现盈亏 | 累计已实现 |
| `cTime` | 开仓时间 | API时间戳 |
| `uTime` | 最后更新 | API时间戳 |

---

**操作记录筛选（严格）：**

OKX 账单 API 返回所有逐仓操作，需严格筛选当前仓位：

| 筛选条件 | 说明 |
|---------|------|
| `posId` 匹配当前仓位 | 只记录本仓位的操作 |
| `ts` ≥ `cTime`（开仓时间） | 从开仓开始，不含历史仓位 |
| `instId` = BTC-USDT-SWAP | 限定币种 |
| `mgnMode` = isolated | 限定逐仓 |

---

### 4. 操作历史记录

**每个仓位包含 `操作记录` 数组，记录从开仓开始的操作事件：**

```json
{
  "操作记录": [
    {
      "时间": "1776397601007",
      "类型": "开仓",
      "价格": "74767.8",
      "数量": "0.01",
      "手续费": "-0.01121462"
    },
    {
      "时间": "1776420000000",
      "类型": "资金费结算",
      "金额": "0.0001"
    },
    {
      "时间": "1776450000000",
      "类型": "止盈触发",
      "价格": "76000",
      "数量": "0.005",
      "盈亏": "0.1"
    }
  ]
}
```

**事件类型：**

| 类型 | 说明 | 数据来源 |
|------|------|---------|
| 开仓 | 首次入场 | bills (type=2) |
| 加仓 | 增加持仓 | bills (type=2) |
| 减仓 | 减少持仓 | bills (type=2) |
| 止盈触发 | TP订单触发 | algo订单 |
| 止损触发 | SL订单触发 | algo订单 |
| 资金费结算 | 8小时结算 | bills (type=8) |

---

**筛选规则：**

OKX 账单 API 返回所有逐仓操作记录，需严格筛选：

| 筛选条件 | 说明 |
|---------|------|
| `posId` 匹配当前仓位 | 只记录本仓位的操作 |
| `ts` ≥ 开仓时间 | 从开仓开始，不含历史仓位 |
| `instId` = BTC-USDT-SWAP | 限定币种 |

**正确做法：**

1. 从当前持仓获取 `开仓时间` 和 `持仓ID`
2. 筛选账单：`持仓ID` 匹配，`时间` ≥ `开仓时间`，`合约` = BTC-USDT-SWAP，`保证金模式` = isolated

**此任务只从 API 获取数据并记录，不做任何事件推断。**

### 5. 日志记录

```bash
NOW=$(date '+%Y-%m-%d %H:%M:%S')

# 记录同步结果
if [ 有BTC逐仓持仓 ]; then
  echo "[$NOW] [阶段一] 实盘持仓同步: BTC逐仓 X 个 (活跃 X, 已平仓 X)" >> logs/daily-report-process.log
else
  echo "[$NOW] [阶段一] 实盘持仓同步: 无BTC逐仓持仓" >> logs/daily-report-process.log
fi

# 记录异常
if [ API获取失败 ]; then
  echo "[$NOW] [阶段一] ⛔ ERROR: 实盘持仓获取失败 - [错误信息]" >> logs/daily-report-process.log
fi
```

### 7. 异常处理

| 异常类型 | 级别 | 处理方式 |
|---------|------|---------|
| OKX API 返回错误 | `⚠️ WARN` | 记录异常，持仓文件标记 `备注: "API获取失败"` |
| 无 BTC 逐仓持仓 | 正常 | 持仓文件 `当前持仓: []` |
| 持仓文件写入失败 | `⛔ ERROR` | 记录异常，继续执行 |

**当实盘无 BTC 逐仓持仓时，根据历史状态分为两种情况：**

---

```json
  },

}
```

---

**情况B：本周期曾开仓已平仓**

```json
{
  "周期ID": "cycle-YYYYMMDD-XXX",
  "同步时间": "YYYY-MM-DDTHH:MM:SS+08:00",
  "数据来源": "OKX实盘账户",
  
  "当前持仓": [],
  
  "最近平仓": {

    "持仓方向": "long",
    "开仓均价": "74767.8",
    "平仓价格": "78000",
    "平仓时间": "1776470000000",
    "平仓类型": "止盈触发",
    "持仓张数": "0.01",
    "盈亏": "0.32",
    "手续费": "-0.011"
  },
  
  "汇总": {
    "当前持仓数": 0,
    "未实现盈亏总计": 0,
    "已实现盈亏总计": 0.32
  },
  
  "备注": "本周期曾开仓现已平仓"
}
```

---

**判断逻辑：**

| 条件 | 结果 |
|------|------|
| 持仓文件不存在 + 无持仓 | 情况A（从未开仓） |
| 持仓文件存在 + 旧持仓为空 + 无持仓 | 情况A（从未开仓） |
| 持仓文件存在 + 旧持仓有仓位 + 无持仓 | 情况B（已平仓） |

**关键点：**
- `当前持仓` 为空数组 `[]`
- `最近平仓` 区分两种情况
- 情况B的 `汇总.已实现盈亏总计` 记录平仓盈亏

---

## 核心原则

1. **每次覆写**：交易所数据是权威来源
2. **筛选严格**：只同步 BTC-USDT-SWAP 逐仓仓位
3. **中文字段名**：便于阅读理解
4. **只记录当前持仓**：无需历史持仓列表
5. **操作记录严格筛选**：只包含当前仓位从开仓开始的操作
6. **平仓检测**：通过对比旧持仓文件判断
7. **异常容错**：同步失败不影响流程继续
