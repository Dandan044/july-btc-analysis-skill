# 日报任务 - 阶段一：数据获取

此任务为日报工作流的第一阶段，负责周期检测、实盘持仓同步、数据采集和初步挖掘。


---

## 触发方式

- 由定时任务触发（每天 9:00 和 21:00 GMT+8）
- 或由主会话路由调用

---

## 日志文件

**所有阶段共用同一个日报进程日志文件：**

路径：`logs/daily-report-process.log`

格式：追加模式，记录每个阶段的开始、结束、警告、错误。

### 日志异常标识规则

**使用特殊标识区分警告和错误：**

| 级别 | 标识 | 含义 | 示例 |
|------|------|------|------|
| **警告** | `⚠️ WARN` | 不影响流程继续执行，但需关注 | 数据获取延迟、历史报告数量不足 |
| **错误** | `⛔ ERROR` | 可能影响后续阶段，需人工介入 | 数据脚本执行失败、文件创建失败、spawn 失败 |

**区分原则：**

| 场景 | 级别 | 原因 |
|------|------|------|
| 数据脚本返回但部分字段缺失 | `⚠️ WARN` | 数据仍可用，阶段二可判断 |
| 数据脚本执行超时/失败 | `⛔ ERROR` | 无数据无法进行阶段二分析 |
| 周期文件夹创建失败 | `⛔ ERROR` | 无法保存后续文件 |
| 历史报告数量不足（< 预期） | `⚠️ WARN` | 不影响流程，阶段二自行处理 |
| 历史报告路径收集失败 | `⚠️ WARN` | 可标记无历史，继续执行 |
| 数据挖掘报告生成失败 | `⚠️ WARN` | 清单中标记缺失，阶段二可直接分析原始数据 |
| 数据清单 JSON 生成失败 | `⛔ ERROR` | 阶段二无法获知文件路径 |
| spawn 阶段二失败 | `⛔ ERROR` | 流程中断，需人工重启 |

**日志格式：**

```
正常: [时间] [阶段X] 内容
警告: [时间] [阶段X] ⚠️ WARN: 内容
错误: [时间] [阶段X] ⛔ ERROR: 内容
```

### 5. 生成数据挖掘报告

**此步骤产出数据洞察，供阶段二分析师参考。**

---

#### 5.0 角色定位

**你现在是一名加密货币市场的数据分析师，擅长从繁杂的市场数据中挖掘事实、发现关联、标注异常。**

**你的职责边界：**
- 只陈述事实、归纳现象、发现数据特征
- **不做主观判断、不给交易建议、不预测未来走势**
- 让数据说话，让后续分析师做决策

---

#### 5.1 ⭐ 核心约束：不做数据呈现

**阶段二会读取原始 JSON 数据进行分析，因此阶段一的数据挖掘报告必须避免"数据呈现"。**

| 禁止行为 | 原因 |
|---------|------|
| 罗列价格表格 | 阶段二模型会自己读取 JSON |
| 重复数值清单 | 冗余，无价值 |
| 列举 OHLC 数据 | 原始数据已在 JSON 中 |
| 制作指标表格 | 阶段二会直接处理 |

| 应该行为 | 原因 |
|---------|------|
| 提炼数据特征 | 产出洞察而非数据 |
| 标注异常点 | 节省阶段二的注意力 |
| 发现联动关系 | 帮助阶段二快速聚焦 |
| 归纳演变模式 | 从数据中提取规律 |

**核心原则：报告输出的是"发现了什么"，而非"数据是什么"。**

---

#### 5.2 报告格式

**保存路径：** `active/cycle-*/data-context/data-mining-YYYY-MM-DD-HHMM.md`

**命名规则：** 时间精确到分钟，使用触发时间。

**深入、细微、谨慎：** 洞察到数据间存在的关系，报告长度根据你的洞察推理深度决定。

---

#### 5.3 挖掘原则

**时间维度：由远及近，由大到小**

观察数据时，遵循时间递进逻辑，输出演变特征而非数值列表：
- 长期趋势的延续或反转特征
- 中期态势的变化特征
- 近期动态与历史的关系
- 不同时间颗粒度之间的一致或分歧

**颗粒维度：由宏观到微观**

先把握整体态势，再聚焦局部特征，输出模式而非数据：
- 整体格局的特征
- 局部异常的表现
- 分时细节与日线的吻合程度

**联动维度：发现数据间的关联**

寻找数据间的相互关系，陈述现象：
- 价格变动时，哪些指标同步或背离
- 不同指标是否指向同一方向
- 存在哪些验证或分歧现象

---

#### 5.4 ⛔⛔⛔ 挖掘要求（严格遵守）

**阶段一绝对禁止的输出类型：**

| 禁止类型 | 错误示例 | 为什么禁止 |
|---------|---------|-----------|
| **预测性语言** | "可能预示变盘" | 这是判断，不是事实 |
| **提示分析方向** | "情绪是否滞后于价格" | 这是给阶段二的暗示 |
| **推断性语言** | "表明多空分歧加剧" | 这是因果判断 |
| **疑问式提示** | "价格是否向痛点移动" | 引导分析思路 |
| **可能性推断** | "存在磁吸效应可能" | 预测性描述 |
| **关注提示** | "关注：是否延续回调" | 提示分析重点 |

**正确做法：只陈述数据本身的现象**

| 错误示例 | 正确示例 |
|---------|---------|
| "可能预示变盘" | "持仓量增加 11.1%，价格下跌 1.87%" |
| "情绪是否滞后于价格" | "FGI 连续 4 日低于 25，价格上涨 4.24%" |
| "表明多空分歧加剧" | "持仓量增幅为 14 日最大值" |
| "价格是否向痛点移动" | "最大痛点距现价 -8.6%，剩余 9 天到期" |
| "存在磁吸效应可能" | "最大痛点 $67,500，现价 $73,872" |
| "关注：是否延续回调" | "当前价格位于 14 日区间 83.4% 分位" |

**核心原则：**

```
错误：陈述数据 + 提示意义/方向/可能性
正确：陈述数据本身的现象，到此为止
```

**动态挖掘，不固化框架：**

根据数据本身呈现的特征，动态决定输出哪些洞察：
- 异常波动 → 输出波动特征（如"单日振幅 15.8%"）
- 走势平稳 → 输出平稳状态（如"连续 3 日涨跌幅 < 1%"）
- 指标极值 → 标注极值位置（如"多空比 1.48 为 14 日峰值"）
- 数据平淡 → 陈述平淡本身（如"各指标未出现显著偏离"）

**切记：输出到此为止，不要添加任何"关注"、"可能"、"预示"等提示性语句。**

**简要说明（不超过 2 行）：**

- 数据源是否正常返回？
- 是否有字段缺失或时效问题？

## 异常处理

**任何步骤发生异常，根据级别记录日志：**
以下是示例

| 异常类型 | 级别 | 处理方式 |
|---------|------|---------|
| 数据脚本执行失败 | `⛔ ERROR` | 清单标记 `status: "failed"`，继续执行（后续阶段可能无法进行） |
| 数据部分字段缺失 | `⚠️ WARN` | 清单标记 `status: "partial"`，注明缺失字段，继续执行 |
| 周期文件夹创建失败 | `⛔ ERROR` | 记录异常，尝试继续执行（后续文件无法保存） |
| 历史报告路径收集数量不足 | `⚠️ WARN` | 清单中记录实际数量，继续执行 |
| 数据挖掘报告生成失败 | `⚠️ WARN` | 清单中标记挖掘报告缺失，阶段二可直接分析原始数据 |
| 数据清单 JSON 生成失败 | `⛔ ERROR` | 记录异常后结束本阶段 |

**不因警告中断流程，错误视情况决定是否继续。**

## 核心要求

1. **首先记录阶段开始**：日志优先，先记录再执行
2. **只检测周期**：创建文件夹，不创建其他文件
3. **同步实盘持仓**：获取OKX实盘数据，筛选BTC逐仓，覆写持仓文件
4. **持仓文件只记录当前持仓**：无历史持仓，操作记录只包含当前仓位的操作
5. **只收集历史报告路径，不读内容**：阶段二自行读取
6. **数据存放在子文件夹**：`data-context/` 与 `reports/` 分离
7. **文件命名带时间信息**：便于区分生成时间
8. **必须生成数据清单 JSON**：固定格式，包含所有必要信息供阶段二读取
9. **Spawn 消息简洁**：只传递数据清单路径，其他信息都在清单中
10. **完成后继续阶段二**：输出数据清单路径，然后读取 tasks/daily-report-stage2.md 继续执行
11. **最后记录阶段结束**：Spawn 完成后记录
12. **异常分级记录**：`⚠️ WARN` 不中断，`⛔ ERROR` 视情况处理

## 阶段过渡

输出数据清单路径后，立即读取并执行 `tasks/daily-report-stage2.md`。数据清单已包含周期、持仓、数据文件、历史报告等全部上下文，阶段二读取清单即可获取所有参数。
