# 日报任务 - 阶段二：数据分析

此任务为日报工作流的第二阶段，负责核心分析和报告生成。

---

## 触发方式

- 由阶段一 spawn 触发
- 接收阶段一传递的数据清单文件

---

## 日志文件

**所有阶段共用同一个日报进程日志文件：**

路径：`logs/daily-report-process.log`

格式：追加模式，记录阶段二的开始、结束、警告、错误。

**异常标识规则：**

| 级别 | 标识 | 含义 |
|------|------|------|
| **警告** | `⚠️ WARN` | 不影响流程继续执行 |
| **错误** | `⛔ ERROR` | 可能影响后续阶段 |

---

## 执行步骤

### 1. 记录阶段开始

**首先在日志中记录本阶段开始：**

```bash
NOW=$(date '+%Y-%m-%d %H:%M:%S')
echo "[$NOW] [阶段二] 开始执行" >> logs/daily-report-process.log
```

---

### 2. 解析 Spawn 消息或定位默认路径

**⚠️ 优先从 spawn 消息解析 data-manifest 路径，若解析失败则从本地默认路径查找（保底措施）。**

#### 2.1 从 Spawn 消息解析参数

预期 Spawn 消息格式：
```
阶段一数据获取已完成。
数据清单: active/cycle-YYYYMMDD-XXX/data-context/data-manifest-YYYY-MM-DD-HHMM.json
请读取 tasks/daily-report-stage2.md 开始阶段二分析。
```

**只需提取数据清单路径，其他所有信息都在清单 JSON 中。**

#### 2.2 读取数据清单获取完整参数

**从 data-manifest.json 获取所有必要信息：**

清单包含的字段：
- `cycle.id` - 周期ID
- `cycle.status` - 周期状态
- `cycle.positions_file` - 持仓文件路径
- `data_files.market_data.path` - 市场数据路径
- `data_files.data_mining_report.path` - 数据挖掘报告路径
- `history_reports` - 历史报告路径列表

#### 2.3 保底措施：从本地默认路径查找

**如果 spawn 消息解析失败（如消息格式异常），执行保底查找：**

```bash
# 查找最新周期
CYCLE_DIR=$(ls -td active/cycle-* 2>/dev/null | head -1)

# 查找最新数据清单
MANIFEST_FILE=$(ls -t ${CYCLE_DIR}/data-context/data-manifest-*.json 2>/dev/null | head -1)
```

**保底日志记录：**
```
[$NOW] [阶段二] ⚠️ WARN: Spawn 消息解析失败，使用保底路径查找
[$NOW] [阶段二] 保底路径: 清单=${MANIFEST_FILE}
```

#### 2.4 确认路径有效性

**读取清单后确认关键路径存在：**

| 路径类型 | 来源 | 失败处理 |
|---------|------|---------|
| 市场数据 | `data_files.market_data.path` | ⛔ ERROR，无法继续 |
| 持仓文件 | `cycle.positions_file` | ⚠️ WARN，假设无持仓 |
| 数据挖掘报告 | `data_files.data_mining_report.path` | ⚠️ WARN，跳过挖掘报告 |

**日志记录：**
```
[$NOW] [阶段二] 数据清单读取: data-context/data-manifest-YYYY-MM-DD-HHMM.json
[$NOW] [阶段二] 周期ID: cycle-YYYYMMDD-XXX
```

### 3. 读取输入数据

**按数据清单中的路径依次读取：**

#### 3.1 读取市场数据

读取 `data/YYYY-MM-DD.json`，获取完整的市场数据。

#### 3.2 读取数据挖掘报告（可选参考）

读取 `data-context/data-mining-*.md`，作为分析参考。

**注意：** 数据挖掘报告仅陈述事实，不做判断。你需要在此基础上进行主观分析和决策。

#### 3.3 读取历史报告内容

按清单中的 `history_reports` 数组，依次读取历史报告：
- 日报文件（btc-report-*.md）
- 即时分析报告（instant-report-*.md）

#### 3.4 读取持仓文件

读取 `positions.json`，了解实盘持仓状态：
- 当前是否有持仓？持仓方向（做多/做空）？
- 入场价、当前盈亏、盈亏比例？
- 止盈止损订单是否已设置？价位是多少？
- 操作记录显示什么？（开仓、加减仓、资金费结算等）

### 5. 分析数据并生成报告

   你是一个专业、果断的比特币分析师。你的分析报告会被后续程序读取并 执行交易动作。

   ### 表述原则

   - 行情分析：基于数据给出你的判断
   - **动作指令：必须明确、可执行**

   ### 动作指令的要求

   对于可执行的内容必须是确定的、可执行的，例如是否入场、是否加、减、平仓、关注什么...,指令必须是明确、可执行。不要使用“建议”等模糊的表述。

   **阶段三会读取你的报告执行动作。动作指令必须可以直接执行，不允 许"考虑"、"建议"、"可以"。**

**结合历史报告和当前周期状态进行本次分析**，按照下方报告结构撰写。


## 报告结构

你的报告必须包含以下六个部分，按顺序呈现：

### 一、历史回顾（如有历史报告）

**此部分必须先完成，用于指导后续分析！**

回顾当前周期内的历史报告：
- 上次报告的预期走势是否应验？
- 上次标注的支撑/压力位是否有效？
- 上次给出的操作建议是否执行？结果如何？
- 还能得出什么结论...

列出参考的报告文件名。

如果没有历史报告（新周期），说明"新周期开始，无历史报告"。


### 二、数据呈现

呈现当前市场的核心数据，让读者快速了解市场状态。用专业的方式展示价格、情绪、技术指标等关键数据。
- 关注数据中的OHLC数据，最高点、最低点是否突破，结合开盘价收盘价是否撤回等，按照时间序列得到走势信息。
- 关于恐慌贪婪指数：其为日级别的更新，因此一日间的恐慌指数保持不变是正常行为，但是依旧能作为跨日级别的判断依据。**注意：不要使用日内恐慌指数不变作为判断依据，恐慌指数的更新跨度为日，不变不是因为情绪没有发生改变，而是因为他没有更新**

### 三、市场多维分析

这是报告的核心。从以下维度审视市场，发现哪个维度有信号就深入哪个，不必面面俱到：

- **价格结构维度：** 价格行为本身揭示了什么形态、节奏、关键位置？
- **动能与趋势维度：** 各周期趋势是否一致？动能是在增强还是衰减？
- **量价与参与维度：** 成交量、持仓量、资金流动揭示了什么市场行为？
- **市场参与者维度：** 不同参与者（散户、大户、主动交易者）的行为是否一致？存在什么分歧？
- **清算与流动性维度：** 清算事件揭示了什么流动性结构？这些位置对后续价格有什么意义？
- **衍生品与情绪维度：** 期权市场、资金费率、情绪指标传达了什么样的市场预期？

对这些维度进行交叉分析，发现矛盾、确认或背离。把看似无关的数据联系起来。

- **为什么会这样？** 近期价格走势背后的驱动因素是什么？
- **是什么导致的？** 哪些数据揭示了当前的市场状态？
- **未来可能发生哪些走向？** 基于数据，你认为市场可能的几种演变路径？
- **各走向的概率？** 给出你的概率评估，并说明依据。

用连贯的专业分析文字，引用具体数据来支撑你的判断。

### 四、行情推断

- **关键位置：** 哪些价格位置是关键支撑/压力？为什么？
- **值得关注的行为：** 后续价格或指标出现什么行为需要特别关注？例如回踩、突破、震荡？这些行为意味着什么？

### 五、仓位操作建议

**首先说明当前实盘持仓状态：**

从 `positions.json` 获取实盘数据：
- 当前是否有持仓？（无持仓 → 观望或开仓建议）
- 持仓方向：做多/做空
- 入场价、当前盈亏、盈亏比例
- 已设置的止盈止损价位（如有）
- 委托订单状态（止盈止损是否已挂单）

---

**给出操作建议：**

根据实盘持仓状态 + 分析结论，给出明确操作指令：

**⚠️ 方向判断：**

1. 当前是否存在利多条件？为什么？哪些数据支撑？
2. 当前是否存在利空条件？为什么？哪些数据支撑？
3. 哪个方向更明确？

**决策原则：**
- 利多条件更明确 → 做多
- 利空条件更明确 → 做空
- 两者都不明确 → 观望

---

**操作建议表格：**

| 项目 | 内容 |
|------|------|
| 操作类型 | 开仓/加仓/减仓/平仓/设置止盈止损/调整止盈止损/观望 |
| 方向 | 做多/做空（开仓时填写） |
| 入场位置 | $xxx（开仓/加仓时填写） |
| 入场条件 | 立即以当前价格入场/或在这里写明相关入场条件，例如你需要确认相关市场数据、价格到达多少、回踩情况、回调幅度、支撑测试等情况 |
| 仓位 | xx% (75%到300%) |
| 止损 | $xxx (-xx%) |
| 止盈1 | $xxx (+xx%，平仓xx%) |
| 止盈2 | $xxx (+xx%，平仓剩余) |
| 盈亏比 | 1:xx（开单盈亏比必须大于1.5，最大亏损：最大盈利） |
| 风险 | 高/中/低 |

---

在表格下方，简要说明这个建议与你分析的关联——为什么给出这个建议？基于哪些数据判断？

若不选择立刻入场，在此处写明白需要关注的市场数据，以决定下一次触发分析所需的条件。
