# 日报任务 - 阶段四：警报管理

此任务为日报工作流的第四阶段，负责警报规则的全生命周期管理。

---

## 触发方式

- 由阶段三 spawn 触发
- 接收标准化的周期状态信息

---

## 日志文件

**所有阶段共用同一个日报进程日志文件：**

路径：`logs/daily-report-process.log`

格式：追加模式，记录阶段四的开始、结束、警告、错误。

**异常标识规则：**

| 级别 | 标识 | 含义 |
|------|------|------|
| **警告** | `⚠️ WARN` | 不影响流程继续执行 |
| **错误** | `⛔ ERROR` | 可能影响后续阶段 |

---

## 执行步骤

### 步骤1: 解析 Spawn 消息或定位默认路径

**⚠️ 优先从 spawn 消息解析参数，若解析失败则从本地默认路径查找（保底措施）。**

#### 1.1 从 Spawn 消息解析参数

预期 Spawn 消息格式：
```
阶段三仓位管理已完成。
周期状态: [所有仓位平仓，已完成归档/周期活跃中]
周期路径: [已归档 | active/cycle-xxx]
请读取 tasks/daily-report-stage4.md 开始阶段四警报管理。
```

**提取关键参数：**

| 参数 | 消息中的标记 | 用途 |
|------|-------------|------|
| 周期状态 | `周期状态:` | 决定执行逻辑（清零 vs 正常管理） |
| 周期路径 | `周期路径:` | 定位报告文件（仅 active 状态需要） |

**说明：** 持仓状态、最近平仓不影响阶段四流程，不传递。

#### 1.2 保底措施：从本地默认路径查找

**如果 spawn 消息解析失败，执行保底查找：**

```bash
# 尝试查找活跃周期
CYCLE_ACTIVE=$(ls -td active/cycle-* 2>/dev/null | head -1)
```

**保底日志记录：**
```
[$NOW] [阶段四] ⚠️ WARN: Spawn 消息解析失败，使用保底路径查找
[$NOW] [阶段四] 保定路径: 周期=${CYCLE_DIR}, 状态=${CYCLE_STATUS}
```

#### 1.3 确认路径有效性

| 路径类型 | 确认方式 | 失败处理 |
|---------|---------|---------|
| 周期目录 | `ls -d $CYCLE_DIR` | ⛔ ERROR，无法继续 |
| 日报文件（active状态） | `test -f $REPORT_FILE` | ⚠️ WARN，跳过报告读取 |

**注意：** archived 状态下不需要日报文件（直接执行警报清零）。

**日志记录：**
```
[$NOW] [阶段四] 参数来源: [spawn解析 | 保底查找]
[$NOW] [阶段四] 周期状态: [active | archived]
[$NOW] [阶段四] 周期路径: [active/cycle-xxx | archived/cycle-xxx]
```

**日志记录（确认解析结果）：**
```
[$NOW] [阶段四] Spawn解析: 成功 / ⚠️ 使用保底
```

### B.4 罗列所有有价值的警报候选

**这个步骤是「发散」阶段：通读日报，尽可能收集所有值得设立的警报。**

**从日报中提取三类警报候选：**

| 警报类型 | 说明 |
|---------|------|
| **上方价格警报** | 阻力位突破、入场触发价格（向上方向） |
| **下方价格警报** | 支撑位跌破、入场触发价格（向下方向） |
| **非价格警报** | 交易量、波动率、持仓量、多空比、资金费率等 |

**不要局限于固定来源，而是从整篇报告中理解并提炼。**

日报的多个部分都可能提供警报线索：
- 价格行为分析 → 趋势判断、关键位置
- 行情推断 → 支撑/阻力位、值得关注的行为
- 仓位操作建议 → 入场条件、触发价格

**示例输出（候选列表）：**

```
候选警报列表（共 N 个）：

上方价格警报候选：
  - xxx
  - xxx

下方价格警报候选：
  - xxx
  - xxx

非价格警报候选：
  - xxx
  - xxx
```

**⚠️ 何时设定非价格类警报？**

每次警报管理时，必须主动评估是否需要设定此类警报。判断依据：

| 市场现象 | 可能的警报类型 |
|---------|---------------|
| 交易量异常放大或萎缩 | 交易量异动警报 |
| 价格在窄幅区间长时间震荡 | 波动率收窄警报（可能预示突破） |
| 短时间内剧烈波动 | 振幅警报 |
| 连续多根同向K线 | 趋势加速警报 |
| 持仓量OI快速增加或减少 | 持仓量变化警报 |
| 多空比发生明显反转 | 多空比反转警报 |
| Taker买卖比偏离常态 | Taker买卖比警报 |
| 资金费率异常（过高或过低） | 资金费率警报 |
| ... | **发散思考，不限于上述示例** |

**⚠️ 重要：发散思维**

以上仅为示例，**切勿局限于这些场景**。每次分析时，应主动思考：

- 当前市场有什么**异常或值得关注的现象**？
- 这些现象如果**持续或反转**，会对持仓产生什么影响？
- 是否有**其他维度**的数据可以提前预警？

**警示**：不要每次只设定价格警报。如果市场处于低波动期，价格警报可能长期不触发，但交易量、波动率的变化可能在提前发出信号。忽略这些信号可能导致错过关键转折点。

**如果候选列表中只有价格警报，说明思维已固化，需要主动打破。**

**日志记录：**
```
[$NOW] [阶段四] 警报候选: 上方价格 X 个 / 下方价格 X 个 / 非价格 X 个 / 共计 X 个
```

### B.5 筛选警报（符合限制）

**这个步骤是「收敛」阶段：从候选中选出最有价值的警报。**

**警报数量限制：**

| 类型 | 最大数量 | 说明 |
|------|---------|------|
| 上方价格警报 | **1 个** | 阻力位 或 入场触发价（向上） |
| 下方价格警报 | **1 个** | 支撑位 或 入场触发价（向下） |
| 非价格警报 | **≤2 个** | 最值得关注的 |
| **总计** | **3~4 个** | |

**筛选原则：**

| 类型 | 筛选逻辑 |
|------|---------|
| **上方价格警报** | 选最接近当前价格且仍有意义的 1 个 |
| **下方价格警报** | 选最接近当前价格且仍有意义的 1 个 |
| **非价格警报** | 选最值得关注/最可能触发的 ≤2 个 |

**示例筛选**（当前价格 $75,000）：

```
上方候选：$78,000 vs $76,000
  → $76,000 更接近当前价 → 选中

下方候选：$73,000 vs $72,000
  → $73,000 更接近当前价 → 选中

非价格候选：交易量、波动率、多空比（3个）
  → 选最值得关注的 2 个

最终警报列表（4个）：
  1. xxx（上方价格）
  2. xxx（下方价格）
  3. xxx（非价格）
  4. xxx（非价格）
```

**日志记录：**
```
[$NOW] [阶段四] 筛选结果: 上方价格 1 个 / 下方价格 1 个 / 非价格 X 个 / 共计 X 个
[$NOW] [阶段四] 最终警报: [警报描述列表]
```

### B.6 执行警报管理

**对比现有规则 vs 最终列表，执行增删改。**

**对比逻辑：**

| 对比结果 | 操作 |
|---------|------|
| 现有规则在最终列表中 | 保持不变 |
| 现有规则不在最终列表中 | 归档 |
| 最终列表中有新警报 | 创建规则文件 |

**创建警报规则时：**

**查看`tasks/set-alert.md`进行警报设立：**

读取 `tasks/set-alert.md`，按其规范创建警报规则文件。

set-alert.md 提供完整的警报规范：
- 规则文件结构（name, interval, check, collect, trigger, lifetime）
- 冷却机制（必须 ≥ 1 小时）
- 异步触发（必须使用 spawn）
- 规则文件命名：`YYYY-MM-DD-<类型>-<描述>.js`
- 可用数据源和 API endpoint

**日志记录：**
```
[$NOW] [阶段四] 保持规则: xxx.js
[$NOW] [阶段四] 归档规则: xxx.js | 原因: 不在最终列表
[$NOW] [阶段四] 创建规则: xxx.js | 目标: xxx
```

**警报管理完成汇总日志：**
```
[$NOW] [阶段四] 警报管理完成 | 活跃规则: X 个 | 归档规则: X 个 | 本次创建: X 个
```

## 异常处理

| 异常类型 | 级别 | 处理方式 |
|---------|------|---------|
| 报告文件读取失败 | `⛔ ERROR` | 记录异常，无法继续执行 |
| 警报规则文件创建失败 | `⚠️ WARN` | 记录异常，继续执行其他警报 |
| 规则归档移动失败 | `⚠️ WARN` | 记录异常，继续执行 |
| 现有规则读取失败 | `⚠️ WARN` | 假定为空，继续执行 |

## 核心要求

1. **⭐ 优先解析 spawn 消息**：从阶段三传递的消息中获取周期状态和路径
2. **保底措施必须执行**：spawn 解析失败时，从本地默认路径查找
3. **周期状态决定流程**：archived → 清零，active → 正常管理
4. **通读整篇日报**：理解分析逻辑，提炼警报意图（仅 active 状态）
5. **归档失效规则前置**：先清理，再规划（仅 active 状态）
6. **两步筛选**：发散罗列候选 → 收敛筛选最终（仅 active 状态）
7. **遵守数量限制**：上方价格 1 个、下方价格 1 个、非价格 ≤2 个
8. **发散思维**：候选阶段必须主动思考非价格维度
9. **禁止 FGI 触发**：恐惧贪婪指数不适合分钟级警报
10. **日志记录完整**：所有操作记录到 daily-report-process.log

## Spawn 消息规范

**阶段三 → 阶段四的 Spawn 消息只需：**

| 参数 | 必须性 | 示例 |
|------|-------|------|
| 周期状态 | ✅ 必须 | `active` 或 `archived` |
| 周期路径 | ✅ 必须 | `active/cycle-xxx` 或 `archived/cycle-xxx` |

**不传递的参数：**
- 持仓状态、最近平仓 → 不影响阶段四流程

**缺失影响：**
- 缺少周期状态 → 无法决定清零还是正常管理（保底可从路径推断）
- 缺少周期路径 → 无法定位报告文件（保底可查找周期目录）

---

## 保底措施说明

**保底触发条件：**
- Spawn 消息格式异常（无法解析）
- Spawn 消息缺少关键参数

**保底查找规则：**

| 参数 | 查找逻辑 |
|------|---------|
| 活跃周期 | `ls -td active/cycle-* 2>/dev/null | head -1` |
| 归档周期 | `ls -td archived/cycle-* 2>/dev/null | head -1` |
| 周期状态 | 有活跃周期 → `active`；有归档周期 → `archived` |
| 日报文件 | `${CYCLE_DIR}/reports/btc-report-*.md`（仅 active 状态需要） |

**保底日志记录：**
- 必须记录 `⚠️ WARN: Spawn 消息解析失败，使用保底路径查找`
- 必须记录保底推断的周期状态

## ⚠️ 执行前自查清单

**在完成警报管理后，必须逐项核对：**

### 数量限制核对

| 检查项 | 要求 | 易错点 |
|-------|------|-------|
| ☐ 上方价格警报是否 ≤ 1 个？ | 选最接近当前价的 1 个 | 误设多个上方警报 |
| ☐ 下方价格警报是否 ≤ 1 个？ | 选最接近当前价的 1 个 | 误设多个下方警报 |
| ☐ 非价格警报是否 ≤ 2 个？ | 选最值得关注的 ≤2 个 | 超过限制 |
| ☐ 总计是否 3~4 个？ | 符合限制 | 警报过多 |

### 思维发散核对

| 检查项 | 正确做法 | 易错点 |
|-------|---------|-------|
| ☐ 候选阶段是否有发散思维？ | 主动评估交易量、波动率等维度 | 只盯着价格 |
| ☐ 候选列表是否只有价格警报？ | 必须有非价格候选 | 思维固化 |
| ☐ 触发条件是否有数据源支持？ | 参考 set-alert.md 可用 API | 无法获取数据 |

### 规则文件核对

| 检查项 | 正确做法 | 易错点 |
|-------|---------|-------|
| ☐ 冷却时间是否 ≥ 1 小时？ | COOLDOWN_MS = 60 * 60 * 1000 | 设定过短 |
| ☐ 是否使用异步触发？ | spawn 创建隔离会话 | 使用 execSync 阻塞 |
| ☐ lifetime 是否合理？ | 根据分析设定有效期 | 永久有效或立即过期 |

### 禁止事项核对

| 检查项 | 正确做法 | 易错点 |
|-------|---------|-------|
| ☐ 是否使用了 FGI 触发？ | ❌ 禁止 | FGI 更新频率为日级 |
